# Vercel 環境変数：正しい値の一覧

**このファイルだけ見れば、何をどこに入れればいいかわかります。**

---

## 本番URLの確認（いちばん重要）

1. Vercel で **kintai-app** を開く
2. 上タブ **「Settings」** → 左 **「Domains」** を開く
3. **Production** のドメインを確認する  
   → 通常は **`kintai-app-khaki.vercel.app`** のような形式  
   → **正しい AUTH_URL / NEXTAUTH_URL の値** = `https://` + そのドメイン（末尾に `/` なし）

例：Domains に `kintai-app-khaki.vercel.app` と出ていれば  
→ **`https://kintai-app-khaki.vercel.app`** が正しい値です。

---

## Vercel に設定する 5 つ（値の取り方）

| 変数名 | 値はどこから取るか | 正しい例・注意 |
|--------|---------------------|----------------|
| **DATABASE_URL** | Supabase の **Connection string**（Pooler, port **6543**） | `postgresql://postgres.xxx:パスワード@aws-0-ap-northeast-1.pooler.supabase.com:6543/postgres` の形式。値の前後に `"` を付けない。 |
| **DIRECT_URL** | Supabase の **Connection string**（Direct, port **5432**） | `postgresql://postgres.xxx:パスワード@aws-0-ap-northeast-1.pooler.supabase.com:5432/postgres` の形式。値の前後に `"` を付けない。 |
| **AUTH_SECRET** | ローカルの `.env` の `AUTH_SECRET=` の右側をそのままコピー | 32文字以上のランダム文字列。ローカルで動いているなら、そのまま同じ値でOK。 |
| **AUTH_URL** | 上で確認した **本番のURL** | `https://kintai-app-khaki.vercel.app`（Domains に表示されている Production の URL）。**localhost は不可。** |
| **NEXTAUTH_URL** | **AUTH_URL と同じ値** でよい | `https://kintai-app-khaki.vercel.app` |

---

## よくある間違い

- **AUTH_URL / NEXTAUTH_URL が `http://localhost:3002` のまま**  
  → 本番では必ず Vercel の本番URL（`https://kintai-app-khaki.vercel.app` など）にすること。

- **値の前後に `"` を付けている**  
  → Vercel の Value 欄には `"` なしで、`postgresql://...` や `https://...` だけを入れる。

- **プレビュー用の長いURL（`kintai-2jkp18d2p-sakagt3s-projects.vercel.app` など）を使っている**  
  → 本番には **Domains に表示されている短い Production 用のURL** を使うこと。

---

## 値の再確認のしかた

- **DATABASE_URL / DIRECT_URL**  
  → Supabase ダッシュボード → 対象プロジェクト → **Settings** → **Database** → **Connection string** で「URI」をコピー。Pooler が DATABASE_URL、Direct が DIRECT_URL。

- **本番URL**  
  → Vercel → kintai-app → **Settings** → **Domains** の Production のドメインを確認し、`https://` を付けたものが正しい AUTH_URL / NEXTAUTH_URL。

この一覧と「Domains」の表示が一致していれば、環境変数は正しいです。
