# ビルドとデプロイの手順（全部）

## 1. ローカルでビルドする（本番と同じチェック）

ターミナルを開いて、プロジェクトのフォルダに移動してから実行します。

```bash
cd /Users/yudai/Desktop/kintai-app
```

### 1-1. Prisma の型を最新にする（必須）

```bash
npx prisma generate
```

### 1-2. 本番用ビルドを実行

```bash
npm run build
```

- **成功した場合:** 最後に `✓ Compiled successfully` や `Route (app)` の一覧が出て終わります。
- **失敗した場合:** 赤い文字で `Failed to compile` や `Type error:` が出ます。そのメッセージのファイル名と行番号をメモして修正します。

### 1-3. ビルドが通ったらローカルで本番モード確認（任意）

```bash
npm run start
```

ブラウザで http://localhost:3000 を開いて動作確認できます。終わったらターミナルで `Ctrl + C` で止めます。

## 2. 変更を GitHub にプッシュして Vercel にデプロイする

### 2-1. 変更を全部ステージする

```bash
cd /Users/yudai/Desktop/kintai-app
git add -A
```

### 2-2. 状態を確認（任意）

```bash
git status
```

変更したファイル一覧が出ます。

### 2-3. コミットする

```bash
git commit -m "fix: 設定APIの型修正とビルド対応"
```

（`-m` の後のメッセージは自由に変えてOKです。）

### 2-4. GitHub にプッシュする

```bash
git push origin main
```

- ブランチ名が `main` でない場合は、`main` の部分を自分のブランチ名に変えます（例: `master` なら `git push origin master`）。
- 初回や認証が切れている場合は、GitHub のログインを求められます。

### 2-5. Vercel でデプロイを待つ

1. ブラウザで **https://vercel.com** を開いてログインする。
2. プロジェクト一覧から **kintai-app** をクリックする。
3. 上タブの **Deployments** を開く。
4. 一番上に新しいデプロイが並ぶので、**Building** → **Ready** になるまで待つ（だいたい 1〜3 分）。

### 2-6. 本番URLで確認する

- **本番URL:** https://kintai-app-khaki.vercel.app  
- ログイン: https://kintai-app-khaki.vercel.app  
- ダッシュボード: https://kintai-app-khaki.vercel.app/dashboard  

ブラウザで開いて、ログイン・ダッシュボードが問題なく表示されるか確認します。

## 3. GitHub 連携していない場合（Vercel CLI でデプロイ）

### 3-1. Vercel にログイン

```bash
cd /Users/yudai/Desktop/kintai-app
npx vercel login
```

表示される手順に従ってログインします。

### 3-2. 本番デプロイ

```bash
npx vercel --prod
```

初回は「Set up and deploy?」「Which scope?」「Link to existing project?」などと聞かれるので、画面の指示に従います。最後に **Production** の URL が表示されます。

## 4. よくある失敗と対処

| 現象 | 対処 |
|------|------|
| `npm run build` で Type error | 表示されたファイルの行を開き、型エラーを直す。直したらもう一度 `npm run build`。 |
| `git push` で permission や reject | GitHub にログインしているか確認。`git remote -v` でリモートが正しいか確認。 |
| Vercel でビルドが失敗する | Vercel の **Deployments** → 失敗したデプロイ → **Building** のログを開き、同じ Type error やエラーが出ていないか確認。 |
| 本番で 500 や画面が真っ白 | Vercel の **Settings** → **Environment Variables** で、`DATABASE_URL`・`DIRECT_URL`・`AUTH_SECRET`・`AUTH_URL` が本番用に設定されているか確認。 |
