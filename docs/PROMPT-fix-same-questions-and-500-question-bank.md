# 【解決プロンプト】同じ問題しか表示されないバグ & プラン作成時に500問バンクを作る

Cursor の Composer（Cmd + I）に以下を貼り付けて実行してください。

---

## 背景と原因

1. **同じ問題しか表示されない**
   - 問題バンク（QuestionBank）は「このプランを適用して開始する」を押したときだけ作成される。適用していないとバンクが存在せず、毎回モック20問 or LLM都度生成のフォールバックになり、同じ・似た問題ばかり出る。
   - または QuestionBank が DB にあっても、Prisma の `Json` 取得で `Array.isArray(raw)` が false になり `bankList` が空のままフォールバックしている可能性がある。

2. **500問は「適用」時だけ**
   - 500問を生成する API（`generate-question-bank`）とプロンプトはあるが、呼ばれるのは「適用」ボタンのみ。「プラン作成」時には20問のプレビューリストしか作られていない。

---

## 依頼内容（以下を実装してください）

### 1. プラン作成完了時に500問バンクを「作成開始」する

- **「この内容でプラン作成」** を押して API が成功したら、その直後に **`POST /api/ai/generate-question-bank`** を呼び出す（goal, level, planSummary はプラン作成で使った値を使う）。
- 非同期でよい。トーストで「500問バンクを生成しています。完了したらダッシュボードで出題されます。」と表示し、プレビュー表示はブロックしない。
- これにより「適用」を押す前にバンクができ始め、**「適用」を押したときは既存バンクがあれば再利用**し、なければそのタイミングで生成するようにする（既にバンクがあれば generate-question-bank の上書きでよい）。

### 2. ダッシュボード出題は「必ずバンク優先」にし、バンクの読み方を確実にする

- **`src/app/api/ai/today-questions/route.ts`** で、QuestionBank の `questions` を読む処理を修正する。
  - Prisma の `Json` 型が配列で返らない場合に備え、`raw` がオブジェクトの場合は `Object.values(raw)` や、JSON 文字列の場合は `JSON.parse(raw)` で配列に正規化してから `bankList` を組み立てる。
  - バンクが存在し、正規化後の配列長が 1 以上なら、**必ずバンクから出題する**（フォールバックは「バンクが空または存在しない」場合のみにする）。

### 3. バンクが空のときのフォールバックを「同じ問題」にならないようにする

- **モック**（OPENAI_API_KEY なし）の場合: 現在の MOCK_POOL 20問に加え、**リクエストごとに異なるシード**（例: `timestamp + Math.random()` をシードに加える）でシャッフルし、`needNew` 問を選ぶ。可能ならモック問題を 30〜50 問に増やし、ローテーションしやすくする。
- **LLM** の場合: バッチごとに **「バッチ番号・日付・ランダムなテーマ」** をプロンプトに含め、毎回異なる問題が返りやすくする。

### 4. 設定画面の文言で「適用」の意味をはっきりさせる

- プランプレビュー付近に、次のような短い説明を追加する。
  - 「このプランを適用すると、500問の出題バンクが保存され、ダッシュボードで毎回ランダムに問題が出題されます。未適用のままでは出題は仮の20問になります。」

### 5. 動作確認チェックリスト（実装後に確認すること）

- プラン作成 → しばらく待つ → ダッシュボードを開く → 出題数が設定どおりで、問題が前回と違うこと。
- 設定で「このプランを適用して開始する」を押した場合も、ダッシュボードで 500 問バンクからランダムに出題されていること。
- API キーなし環境では、モックでも 20〜50 問のプールからシャッフルされ、できるだけ同じ問題が連続しないこと。
