# Safari でモバイル時にサイドバーが消えない問題

## 現象
スマホ（Safari）でダッシュボードを開くと、画面の半分がネイビーのサイドバーになり、メインが圧迫される。クッキー削除後も変わらない。

## 想定していた原因

1. **Safari の CSS 解釈**
   - Tailwind の `hidden lg:flex` は「通常は `display: none`」「1024px 以上で `display: flex`」になるが、Safari（特に iOS）でメディアクエリや詳細度の扱いが他ブラウザと異なり、意図どおり効かないことがある。

2. **キャッシュ**
   - クッキーだけ削除しても、CSS/JS や HTML はキャッシュされていることがある。Safari や Vercel のキャッシュで古いバンドルが返っている可能性。

3. **SSR とクラス指定**
   - サーバーは「どの幅で見ているか」を知らないため、同じ HTML を返す。`hidden lg:flex` はクライアントの CSS で切り替わるが、Safari でその切り替えが期待どおり動いていない可能性。

## 実施した対策

- **CSS のメディアクエリだけに頼らない**
  - `window.matchMedia("(min-width: 1024px)")` で「PC かどうか」を JS で判定。
  - **1024px 未満（モバイル）**: PC 用サイドバーを **DOM に含めない**（`{isDesktop && (...)}` で描画しない）。  
    → レイアウト上も存在しないため、Safari でも確実に非表示。
  - **1024px 以上（PC）**: これまでどおりサイドバーを表示。
- モバイル用ヘッダー・下ナビ・ドロワーは `isDesktop === false` のときだけ描画するように統一。

## 確認手順（Safari）

1. 最新コードをデプロイする。
2. **ハードリロード**: Safari で「保持しているデータを消す」または「スーパーリロード」を行う。  
   （例: 設定 → Safari → 履歴とWebサイトデータを消去、または開発メニューで「空のキャッシュ」）
3. ダッシュボードを再度開き、画面幅いっぱいにメインだけ表示され、≡ タップでドロワーが出ることを確認する。
